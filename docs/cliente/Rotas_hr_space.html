Rotas — hr-space

Mapa de telas da webview hr-space, utilizada no Espaço RH do app Meu Alelo.

A aplicação utiliza HashRouter, portanto a URL final deve seguir o padrão:

Código
{BASE_URL}/#{ROTA}

Exemplo:

Código
https://meualelo-webviews.alelo.com.br/#/card-tracking

Padrão de navegação via app

Rotas-hr-space.txt · seção 2

Para abrir uma rota da webview dentro do app Meu Alelo, deve ser utilizado o deeplink:

Código
meualelo://app/webview?url={URL_ENCODED}&isModal=false&showNavbar=false&authRequired=true

Onde:

  • url={URL_ENCODED} deve receber a URL completa da webview em formato URL encoded.
  • isModal=false indica que a webview não será aberta como modal.
  • showNavbar=false indica que a navbar nativa não será exibida.
  • authRequired=true indica que o usuário precisa estar autenticado.

Exemplo

URL original:

Código
https://meualelo-webviews.alelo.com.br/#/card-tracking

URL encoded:

Código
https%3A%2F%2Fmeualelo-webviews.alelo.com.br%2F%23%2Fcard-tracking

Deeplink final:

Código
meualelo://app/webview?url=https%3A%2F%2Fmeualelo-webviews.alelo.com.br%2F%23%2Fcard-tracking&isModal=false&showNavbar=false&authRequired=true
03

Uso pelo agente de IA

Rotas-hr-space.txt · seção 3

Quando o agente retornar uma resposta com atalho visual, deve usar o elemento:

Código
[list_navigation](meualelo://app/webview?url={URL_ENCODED}&isModal=false&showNavbar=false&authRequired=true)

O frontend deve identificar o elemento [list_navigation] e renderizar um componente visual para abrir a rota correspondente dentro da webview.

Exemplo:

Código
Encontrei o pedido **342671**.

Status: **Pago**
Data do pedido: **21/05/2026**

[list_navigation](meualelo://app/webview?url={URL_ENCODED_ORDER_DETAIL}&isModal=false&showNavbar=false&authRequired=true)

Rotas recomendadas para uso pelo agente

Rotas-hr-space.txt · seção 5

Consultar colaborador

Quando a resposta estiver relacionada a colaboradores, o agente pode direcionar para:

Código
#/employees

Ou, quando houver identificador disponível:

Código
#/employees/:id/edit

Exemplo conceitual:

Código
[list_navigation](meualelo://app/webview?url={URL_ENCODED_EMPLOYEE_ROUTE}&isModal=false&showNavbar=false&authRequired=true)

Consultar pedido

Quando a resposta estiver relacionada a um pedido específico, o agente pode direcionar para:

Código
#/order-detail/:orderNumber

Exemplo:

Código
https://meualelo-webviews.alelo.com.br/#/order-detail/342671

Consultar colaboradores do pedido

Quando a resposta estiver relacionada aos colaboradores de um pedido, o agente pode direcionar para:

Código
#/order-detail/:orderNumber/beneficiaries

Exemplo:

Código
https://meualelo-webviews.alelo.com.br/#/order-detail/342671/beneficiaries

Consultar últimos pedidos

Quando a resposta estiver relacionada à lista de pedidos, o agente pode direcionar para:

Código
#/orders

Rastrear cartões

Quando a resposta estiver relacionada a rastreamento de cartões, o agente pode direcionar para:

Código
#/card-tracking

Quando houver número do pedido:

Código
#/card-tracking/:orderNumber

Quando houver número do pedido e AR:

Código
#/card-tracking/:orderNumber/:arNumber

Criar pedido

Quando o usuário perguntar como fazer um pedido, o agente pode explicar que não cria pedidos pelo chat e direcionar para a jornada oficial:

Código
#/new-order/products

Regras para montagem das URLs

Rotas-hr-space.txt · seção 6

  1. A rota deve ser montada usando a base URL do ambiente.
  2. Como a aplicação usa HashRouter, a rota deve vir após /#/.
  3. A URL completa da webview deve ser convertida para URL encoded antes de ser enviada no parâmetro url do deeplink.
  4. O deeplink final deve seguir o padrão:
Código
meualelo://app/webview?url={URL_ENCODED}&isModal=false&showNavbar=false&authRequired=true
  1. O agente não deve inventar IDs, números de pedido ou AR.
  2. O agente só deve retornar list_navigation quando a rota estiver diretamente relacionada à resposta.
  3. Links para jornadas transacionais devem apenas abrir a tela oficial, sem executar a ação automaticamente.
