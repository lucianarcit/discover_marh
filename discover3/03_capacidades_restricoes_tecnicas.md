# 03 — Capacidades e Restrições Técnicas

> **Princípio:** Este documento analisa exclusivamente:
> - `docs\cliente\Gestao_de_Colaboradores.html`
> - `docs\cliente\Gestao_de_Pedidos.html`
> - `artifacts\api_inventory\` (JSONs e relatórios)
> - `docs\reports\`
>
> Nenhum item aqui é requisito do cliente. São **fatos técnicos observados, documentados ou inferidos** das APIs e dos testes reais.

---

## Formato de cada item

| Campo | Descrição |
|---|---|
| **TEC ID** | Identificador único (ex.: TEC-001) |
| **Tipo** | API_CAPABILITY / API_LIMITATION / API_DEPENDENCY / SECURITY_CONSTRAINT / DATA_CONSTRAINT / NOT_VALIDATED / OBSERVED_BEHAVIOR / DOCUMENTED_BEHAVIOR / CONFLICT |
| **Capacidade ou restrição** | Descrição factual e objetiva |
| **Fonte** | Arquivo de origem |
| **Evidência** | Trecho ou dado que sustenta o item |
| **Impacto no agente** | Como isso afeta o comportamento ou implementação do agente |
| **Status** | CONFIRMED / AMBIGUOUS / NOT_VALIDATED / CONFLICTING |
